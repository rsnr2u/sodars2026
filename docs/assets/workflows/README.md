# Asset Folder: Workflows

> **This document represents the finalized Version 1 architecture. Any new feature outside Version 1 must be documented under `12-future-roadmap.md` before implementation.**

## Purpose
The purpose of this folder is to store step-by-step logic workflows and state-transition flowchart assets.

## Scope
This folder contains detailed drawings showing how users traverse screens and how the backend state transitions execute across various models.

## Future Contents
In future development phases, this folder will house:
* Provider screen registration and approval state flows.
* Campaign checkout and payment validation workflows.
* Customer creative content approval and rejection cycles.
* Push notification trigger condition trees.
