# Mobile Module: Future Scope

> **This document represents the finalized Version 1 architecture. Any new feature outside Version 1 must be documented under `12-future-roadmap.md` before implementation.**

## Purpose

The purpose of this document is to list future requirements and business features for Mobile applications that are out of scope for Version 1.

---

## Scope

This document specifies:
* Out-of-scope wearable apps targets.
* Future background location utilities and geofencing.
* IoT player control tools.

---

## Business Rules

### 1. Deferred Features (Out of Scope for V1)

* **Wearables Support (Apple Watch & Wear OS)**:
  * Extensions allowing operators to review dashboard notifications and confirm screen listings directly from smartwatches.
* **Offline Maps**:
  * Local caching of vector map tiles so maps function without internet access.
* **Route Optimization for Field Staff**:
  * Using Google Maps directions APIs to auto-calculate the most efficient driving routes for operators checking multiple screen installations.
* **Geofencing & Background Location tracking**:
  * Continuous tracking of worker positions in the background to verify loop checks audits and trigger proximity notifications.
* **Digital Signage Remote Control via Bluetooth / NFC**:
  * Enabling field workers to connect directly to media players via Bluetooth Low Energy (BLE) or NFC tags to upload diagnostic playlists or run manual resets.
* **Drone Camera Integration**:
  * Supporting connection to drone cameras to capture aerial site photos of billboards automatically.
* **Augmented Reality (AR) Previews**:
  * Using smartphone cameras to overlay digital creative layouts onto empty physical billboards in real-time.
