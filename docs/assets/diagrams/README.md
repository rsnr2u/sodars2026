# Asset Folder: Diagrams

> **This document represents the finalized Version 1 architecture. Any new feature outside Version 1 must be documented under `12-future-roadmap.md` before implementation.**

## Purpose
The purpose of this folder is to store architectural, structural, and relationship diagrams for the SODARS project.

## Scope
This folder contains static and dynamic visual diagrams representing the backend, frontend, and system components of SODARS.

## Future Contents
In future development phases, this folder will house:
* UML Class diagrams.
* System Entity-Relationship Diagrams (ERD).
* Sequence diagrams representing core system events (e.g., payment checkout, screen verification).
* Network deployment topology diagrams.
